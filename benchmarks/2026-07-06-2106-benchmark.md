## Changes (root-caused, defaults-only â€” no benchmark tuning, no URL special-casing)

### Fix A â€” scoring coverage guard (layer: Rust core, `selector/content.rs`)

- **Root cause.** On pages whose content is thousands of `<p>` directly under `<body>`
  (no article/main wrapper), `find_by_scoring`'s candidate set (`div, section, article,
main, td`) never includes the body, so it latched onto the largest small div â€” on the
  Gutenberg Pride & Prejudice page, one `<div class="blockquot">` letter (1.4% of the book).
  rs-trafilatura avoids this with a coverage guard in its heuristic scorer
  (rs `extract.rs` "coverage < 0.3"): a winner covering under 30% of the body text is
  rejected, and extraction falls back to the whole body.
- **Fix.** Ported the guard: `find_by_scoring` now returns `None` when the winner's text
  covers <30% of the body text (integer math, `MIN_SCORING_COVERAGE_NUM/DEN = 3/10`);
  the cascade then falls through to its body floor. Applies only to the weakest cascade
  stage (scoring), exactly like the reference.
- **Pages fixed.** gutenberg.org P&P **0.028 â†’ 0.999** (1,811 â†’ 128,698 words extracted).
- **Tests.** 3 new (flat-body fragment rejected; â‰¥30%-coverage winner kept; gated
  Gutenberg fixture asserting >400k chars).

### Fix B â€” hidden-element discard (layer: Rust core, `html_processing.rs`)

- **Root cause.** trafilaturacore kept visually-hidden text the reference drops. Canonical
  Trafilatura's default `OVERALL_DISCARD_XPATH` (2nd expression, `xpaths.py:131-140`;
  identical in go-trafilatura) deletes any element with `hidden` in id/style,
  `display:none` / `display: none` in style, hide-ish class tokens (prefix `hide-`, infix
  `-hide-`, `hide-print`, space-led `" hidden"` / `" hide"`, `noprint`, `notloaded`), or
  `aria-hidden="true"`.
  rs-trafilatura lacks this rule (it kept the same junk). Concrete case: 50Ã—
  `<span style="display: none;">Unsupported:</span>` screen-reader spans in Wikipedia
  release-history tables.
- **Fix.** Ported exactly those hidden-element conditions as a whole-subtree removal pass
  in `clean_document` (before the strip/clean lists), with a backoff that restores the
  snapshot if the pass removes every `<p>` (protects pages that hide their whole body).
- **Pages fixed.** wikipedia TypeScript **0.905 â†’ 0.936**, Web_scraping +0.006,
  World_Wide_Web +0.006, wikinews +0.007, css-tricks +0.015, go.dev/blog/strings +0.025.

## Explanations (what these numbers mean)

- **Fidelity is reference-relative.** Token precision/recall/F1 are measured against
  Trafilatura's output, not human-labelled gold (none exists for arbitrary live pages).
  They quantify agreement with the canonical extractor â€” not correctness.
  rs-trafilatura is a comparison peer, not a target to overfit.
- **The engines do different jobs.** trafilaturacore returns sanitized, formatted **HTML**
  plus a page-type classification (strictly more work); the other three return plain
  text. Outputs are normalized to text for scoring, so **speed and output size are not
  like-for-like** â€” trafilaturacore's time is its real user-facing cost, not an extraction
  kernel benchmark.
- **Speed is internal extraction time** after one untimed warm-up per engine, in a
  single long-lived process (isolates algorithm speed from startup/model load).
- **Precision vs recall.** High precision = clean output (little boilerplate kept);
  high recall = complete output (little real content missed); F1 balances the two.
- **Defaults only.** Every engine runs with default options; no per-page or per-corpus
  tuning; no URL is special-cased. Both fixes are faithful ports of reference behavior
  (rs-trafilatura's coverage guard; Trafilatura/go-trafilatura's hidden-element rule)
  that apply to every page equally.
