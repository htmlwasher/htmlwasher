## Changes

**None.** This iteration applied no code fix. Each worst-page gap was root-caused to a
reference-relative artifact or per-page noise, not a defaults-only-fixable defect. Per the
non-negotiable rules ("a fix that only moves the benchmark is a regression in disguise";
"prefer robustness fixes that help ALL similar pages"), tuning any of them would degrade real
quality or overfit the corpus:

- **overreacted.io (article, gap +0.109) — reference drops code, tc keeps it.** Token-diff:
  tc emits 1847 tokens the reference omits, almost entirely code (`const` appears **76×** in
  tc vs **1×** in the reference; `useEffect` 67× vs 23×). This is a code-heavy React tutorial
  whose code blocks _are_ the content. Trafilatura aggressively strips them; rs matches
  Trafilatura (7868w). Making tc drop the code to match would delete legitimate content — a
  quality regression to win reference-relative F1. **Documented, not tuned.**

- **webscraper.io all-in-one (product, gap +0.162) — degenerate reference.** The Trafilatura
  reference here duplicates its own intro paragraph and drops the product cards; tc keeps the
  intro once and the product listings (specs, prices, review counts). tc is arguably more
  correct. Detected `service` (expected `product`), but the page is a training fixture whose
  "right" answer is ambiguous, and it is 46–82 words — F1 noise. **Documented, not tuned.**

- **books.toscrape travel_2 (collection, gap +0.135) — per-page noise, not systematic.** tc
  drops the book-grid titles here (56w vs ref 152w). But its sibling category page
  `mystery_3` — identical HTML structure, same detected type (`collection`), same profile and
  link-density path — scores **0.936**. A systematic profile/link-density bug would sink both;
  it does not. Chasing travel_2 in isolation is fitting reference noise. tc also leads rs on
  **5 of 6** collection/listing pages. **Documented, not tuned.**

- **wikinews story_preparation (article, gap +0.051) — reference drops a short list tc keeps.**
  105 extra tokens, all from a short prepared-stories list the reference omits. Minor, and
  keeping it is defensible. **Documented, not tuned.**

- **Rust core / classifier surgery rejected on cost-benefit.** The only structural levers left
  (link-density thresholds; retraining the XGBoost classifier to flip webscraper `service→product`
  or overreacted `documentation→article`) are high-risk changes to a mature core that already
  **leads all comparison engines**, and would jeopardize the 100-page adbar eval oracle
  (F1 0.835) and the Rust↔Python parity fixtures for negative expected value on ≤3-page,
  reference-noisy types. The prior `/bench:improve` run (`2026-07-06-2106`) reached the same
  determination; this run confirms it holds after the cleaning refactor.

## Regression check (cleaning-levels refactor, commit `e7a808d`)

The refactor replaced the five-level preset system with the single `DEFAULT_CLEAN_CONFIG` and
renamed `boilerplate: 'none' → 'clean-only'`. Benchmark impact: overall F1 **0.897 → 0.898**,
precision 0.939 → 0.941, recall stable; per-type F1 unchanged on 5 of 6 types (product +0.016
noise). **No regression** — the refactor is F1-neutral, as expected for a cleaning-API change
that does not alter the Rust extraction or classification path.

## Explanations (mandatory)

- **Fidelity is reference-relative.** Token P/R/F1 measure agreement with **Trafilatura**, not
  human-labelled gold (none exists for arbitrary live pages). A higher score means closer to
  the canonical extractor — not "more correct." rs-trafilatura is a comparison peer, not a
  target to overfit; where the reference itself drops real content (code, product cards), a
  lower F1 can mean _better_ extraction.
- **The engines do different jobs.** trafilaturacore returns sanitized, formatted **HTML** plus a
  page-type classification (strictly more work); Trafilatura, rs-trafilatura, and readability
  return plain **text**. Outputs are normalized to text for scoring, so **speed and output size
  are NOT strictly like-for-like**.
- **Speed is internal extraction time** after one untimed warm-up (isolates algorithm speed
  from process startup / model load).
- **Precision vs Recall.** High precision = clean output (little boilerplate kept); high recall
  = complete output (little real content missed); F1 balances the two. trafilaturacore leads on
  precision (0.941) and F1 while keeping recall competitive (0.901).
- **Defaults only.** No per-page or per-corpus tuning; no URL is special-cased. This run applied
  no fix precisely because every remaining gap fails that bar.
