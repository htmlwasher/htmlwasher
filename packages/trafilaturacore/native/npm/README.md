# @trafilaturacore/native platform prebuilds

Per-target packages holding the committed native binaries for `@trafilaturacore/native`
(contextractor committed-prebuild pattern). The loader (`../index.js`, generated by
`napi build`) `require()`s the matching `@trafilaturacore/native-<target>` at runtime.

## Committed binaries

- **`darwin-arm64/trafilaturacore-native.darwin-arm64.node`** — committed (built on the
  Apple-Silicon host with `pnpm --filter @trafilaturacore/native build:rebuild`).

## CI-produced binaries (NOT committed here — do not fake them)

The other targets need cross-toolchains this host does not have, so CI builds and
commits their binaries:

- `darwin-x64`, `linux-x64-gnu`, `linux-arm64-gnu`, `win32-x64-msvc` — cross-built with
  Zig (`napi build -x` / `cargo-zigbuild`).
- `wasm32-wasi` — built with `pnpm --filter @trafilaturacore/native build:wasm`, which needs
  `WASI_SDK_PATH`. napi regenerates this package.json (adds the `browser`/wasi-worker
  glue) as part of the wasm build; the placeholder here just keeps the workspace + the
  `file:` optionalDependency resolvable.

Each `package.json` declares `os`/`cpu` (+ `libc` for gnu) so npm/pnpm install only the
matching target on a consumer's machine; mismatched optional deps are skipped.
